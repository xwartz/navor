# 核心概念

Navor 记录持久的投资事实，并从这些事实派生视图。

## 文本形态参考 Beancount，不追求兼容

Navor 使用纯文本、带日期的记录、缩进元数据、posting、注释、确定性解析和 Git 审阅。它不解析 `.bean` 文件，也不复刻 Beancount 的会计模型。

`research`、`thesis`、`decision`、`review`、`journal` 和 `note` 是原生的投资过程事实。

## 事实优先于命令

源文件记录已发生的事和作者已作出的决定：不下单、不跑策略，也不把实时报价当作源事实。用新指令扩展历史，不要改写旧记录来掩盖过去的决策。

## 源事实优先于数据增强

`.nav` 文件只保存用户确认的事实。引擎派生持仓、价值、盈亏、配置与偏离度。价格和汇率提供方可增强视图；数据缺失或过期不得阻止工作区被读取或渲染。

Asset 的 `target` 相对其所属 Account。全组合权重由 Account 与 Asset 目标派生，不要作为第二份源事实写入。

## 稳定的公开接口

已文档化的 `.nav` 语言、`nav` CLI、Reader 输出和静态站点格式是公开接口。`@navor/*` 包用于支撑 CLI；除非另有说明，其导入 API 不稳定。

## 兼容性

Navor 尚未到 1.0。破坏性语法或语义变更须提供变更日志、迁移说明、fixture 和诊断。1.0 需对已文档化语言与 CLI 遵守 SemVer。
